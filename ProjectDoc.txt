1. Project Overview
A city-level waste collection vehicle tracking system enabling municipal administrators and citizens to monitor garbage collection vehicles in real time across multiple cities.
1.1 Problem Statement
Citizens have no visibility into when the garbage vehicle will arrive
Municipality has no real-time fleet oversight across the city
No structured route or stop management for waste collection
1.2 Solution
Live GPS tracking of all vehicles on a map
Predefined routes and stops per vehicle, visible to citizens
Admin dashboard for multi-city fleet management
Simple citizen interface â€” enter vehicle code, see live location

4.3 Why city_id is Required
35+ vehicles per city â€” admin needs to filter by city
Multi-city system â€” Lucknow vehicles should not appear in Kanpur dashboard
Vehicle code prefix carries city info â€” LKO = Lucknow, KNP = Kanpur
Can be auto-derived on server from vehicle_id â€” driver does not type it manually

6. System Architecture
6.1 High-Level Flow
Flutter App / GPS Hardware
   |
   | POST /api/location (every 5 seconds)
   v
Node.js Backend (Railway)
   |-- Validate payload
   |-- Save to PostgreSQL (Supabase)
   |-- Broadcast via Socket.io
   v
Socket.io Channels
   |-- admin-room       (all 35 vehicles)
   |-- vehicle-LKO-001  (single vehicle)
   v
Next.js Frontend (Vercel)
   |-- Admin Dashboard  (all vehicles map)
   |-- Citizen Website  (single vehicle search)
6.2 Socket.io Room Strategy
Each vehicle gets its own room: vehicle-LKO-001, vehicle-LKO-002
Admin joins admin-room and receives all vehicle updates
Citizen joins only the specific vehicle room they searched
Bandwidth optimized â€” citizen does not receive 35 vehicles data
6.3 Multi-City Isolation
Every API request carries city_id
Admin filtered by their assigned city on login
Super Admin sees all cities
Citizen selects city then enters vehicle code
Single server handles all cities â€” no separate deployments

11. Route and Stop Management
11.1 How Routes Work
Each vehicle is assigned exactly one active route
A route has a name and belongs to a city
Each route has multiple stops in order (stop_order: 1, 2, 3...)
Admin creates and manages routes from the dashboard
Citizen sees the route line and all stops on the map
11.2 Stop Information
Stop name (e.g. Gandhi Nagar Chowk, Sector 4 Gate)
Latitude and longitude â€” set by admin by clicking on map
Order number â€” sequence in which vehicle visits
ETA calculated from vehicle current position and speed
11.3 Admin â€” Adding a Stop
Step by step from admin dashboard:
Select city and then select vehicle
Go to Route Builder
Click on map to place stop pin
Enter stop name and confirm order
Save â€” route is live immediately
