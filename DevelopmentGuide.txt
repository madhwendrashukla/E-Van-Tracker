Step 1.3 â€” Create the Database Tables
Go to the SQL Editor in Supabase dashboard and run the following SQL blocks one by one:

Create cities table:
CREATE TABLE cities (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  code TEXT NOT NULL UNIQUE,
  state TEXT NOT NULL,
  created_at TIMESTAMP DEFAULT NOW()
);

Create routes table:
CREATE TABLE routes (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  city_id UUID REFERENCES cities(id),
  name TEXT NOT NULL,
  created_at TIMESTAMP DEFAULT NOW()
);

Create vehicles table:
CREATE TABLE vehicles (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  vehicle_code TEXT NOT NULL UNIQUE,
  city_id UUID REFERENCES cities(id),
  driver_name TEXT,
  route_id UUID REFERENCES routes(id),
  status TEXT DEFAULT 'active',
  created_at TIMESTAMP DEFAULT NOW()
);

Create stops table:
CREATE TABLE stops (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  route_id UUID REFERENCES routes(id),
  name TEXT NOT NULL,
  lat FLOAT NOT NULL,
  lng FLOAT NOT NULL,
  stop_order INTEGER NOT NULL,
  created_at TIMESTAMP DEFAULT NOW()
);

Create location_logs table:
CREATE TABLE location_logs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  vehicle_id UUID REFERENCES vehicles(id),
  city_id UUID REFERENCES cities(id),
  lat FLOAT NOT NULL,
  lng FLOAT NOT NULL,
  speed FLOAT,
  source TEXT DEFAULT 'app',
  timestamp TIMESTAMP DEFAULT NOW()
);

Create users table:
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email TEXT NOT NULL UNIQUE,
  password_hash TEXT NOT NULL,
  role TEXT NOT NULL,
  city_id UUID REFERENCES cities(id),
  created_at TIMESTAMP DEFAULT NOW()
);

Auto-cleanup rule for location_logs (run daily):
DELETE FROM location_logs WHERE timestamp < NOW() - INTERVAL '24 hours';
Note: Set this as a scheduled job in Supabase Dashboard > Database > Cron Jobs.

PHASE 3 â€” Node.js Backend Setup (VS Code)
The backend is the heart of the system. It receives GPS data from the Flutter app, saves it to Supabase, and broadcasts it via Socket.io to the frontend.

Step 3.1 â€” Create Project Folder and Initialize
Open VS Code. Open terminal inside VS Code (Ctrl + ` backtick):

cd E-Van-gaadi-backend
npm init -y

Step 3.2 â€” Install Required Packages

npm install --save-dev nodemon

Purpose

Step 3.3 â€” Create .env File
Create a file named .env in the root of your backend folder:

SUPABASE_ANON_KEY=your_supabase_anon_key_here

IMPORTANT: Never commit this file to GitHub. Add .env to your .gitignore file.

Step 3.4 â€” Create the Main Server File
Create index.js in the backend folder. This file handles three things: REST API endpoint for receiving GPS data, Supabase database writes, and Socket.io broadcasting.

File structure to create:

.env â€” secret keys (never commit to git)
.gitignore â€” exclude node_modules and .env

Step 3.5 â€” Update package.json Scripts

  "dev": "nodemon index.js"
}

Step 4.5 â€” GPS Logic Summary
The core tracking logic works as follows:
On Start Tracking â€” request location permission, start background service
Every 5 seconds â€” get current lat/lng from geolocator
Try to POST to backend API
If POST succeeds â€” coordinate is sent and logged
If POST fails (no internet) â€” save coordinate to SQLite offline buffer
When internet returns â€” flush SQLite buffer to backend in order
On Stop Tracking â€” stop background service, flush remaining buffer

PHASE 6 â€” Deployment

Step 6.3 â€” Deploy Frontend to Vercel
